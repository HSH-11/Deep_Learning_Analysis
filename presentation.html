실제 환경(In-the-wild) 식물 질병 진단

AI 도메인 적응(Domain Adaptation) 연구와 한계 극복기

[최종 결과 보고]

The Problem: 충격적인 도메인 시프트

우리는 단색 배경의 깨끗한 실험실 데이터(PlantVillage)로 완벽한 인공지능을 만들었습니다.

실험실 검증 정확도
99.73%

하지만, 실제 흙과 배경이 섞인 야외 데이터(PlantDoc)를 테스트하자 결과는 처참했습니다.

야외 테스트 정확도
19.12%

Analysis: 왜 실패했을까? (Grad-CAM)

AI가 대체 어디를 보고 판단하는지 뇌 구조를 시각화 해보았습니다.

  • 발견: AI는 잎사귀의 병변을 보지 않고 있었습니다.
  • 오류 원인: 흙, 가지, 배경의 경계선에 집중하며 엉뚱한 식물로 오해(Overfitting)했습니다.

우리는 이것이 Domain Shift (도메인 이동) 현상임을 증명했습니다.

실제는 사과나무인데, 배경을 보고 딸기병으로 오답을 내는 모습

Hypothesis 1: 배경이 문제면 지우면 될까? (V3)

인간의 직관적인 해결책: "전체 이미지의 배경(흙, 가지)을 AI(rembg)로 다 날려버리자!"

  • 2,800여장의 전체 데이터를 누끼 처리 후 재학습(파인튜닝) 시도.
  • 결과: 47.81% (오히려 하락)
실패 증명 (인사이트)

배경을 강제로 날리는 물리적 전처리는 정답을 유추할 '환경적 문맥(Context)'을 파괴하며, 질병으로 썩어 들어간 잎 끝부분까지 훼손시킨다.
V3 학습 곡선 (오버피팅 없이 47%에서 수렴)

Hypothesis 2: 과거 데이터를 섞으면 어떨까? (V4)

또 다른 직관: "기존 깨끗한 데이터(5000장)를 야외 데이터와 섞어서 질병 패턴 망각을 막아보자!"

  • 단계적 파인튜닝, TTA, 혼합 도메인 학습 기법 총동원.
  • 결과: 47.01% (역시 실패)
실패 증명 (인사이트)

모델이 가혹한 야외 환경에 뼈를 깎으며 적응(Adaptation)해야 할 시점에, 쉬운 데이터가 섞여 들어오면서 모델의 시선이 분산되고 안일해졌다.
V4 혼합 학습 곡선 (타겟 도메인 평가에서 47% 한계)

The Solution: 딥러닝 본연의 강건성 (V2)

꼼수(전처리, 데이터 혼합)를 버리고 정공법을 택했습니다.

  • 타겟 도메인(야외 데이터)만을 집중 학습
  • 가혹한 데이터 증강(ColorJitter, RandomCrop)으로 노이즈 내성 강화
  • 수학적 최적화 (Cosine Annealing LR, Label Smoothing)
최종 야외 적응 정확도
60.56%

Conclusion

극한의 Domain Shift 현상을 극복할 때,
인간의 직관에 의존한 물리적 전처리(배경 제거)는 함정입니다.

모델이 가혹한 환경에 노출된 원본 데이터 그 자체를
수학적 최적화와 강건한 증강기법을 통해 스스로 이겨내게 하는 것.
그것이 진정한 도메인 적응(Domain Adaptation) 입니다.

Future Works: 향후 연구 방향

정확도를 80~90% 이상으로 끌어올리기 위한 4가지 고도화 전략을 제안합니다.

  • 1. Vision Transformer (ViT) 도입: 국소적 픽셀에 집착하는 CNN의 한계를 넘어, 글로벌 문맥에 Self-Attention하여 쓸데없는 배경을 무시하고 잎사귀 병변에만 집중.
  • 2. 적대적 도메인 적응 (DANN): 모델 내부에 기울기 반전 계층(Gradient Reversal Layer)을 추가하여, 실험실과 야외 도메인 간의 차이를 모델이 구별하지 못하게 강제함.
  • 3. 객체 탐지(YOLO) 파이프라인 결합: 무식한 배경 제거 대신, 전체 사진에서 '식물의 잎사귀'만 Bounding Box로 잘라낸 뒤 분류를 수행하여 노이즈를 획기적으로 차단.
  • 4. 자기 지도 학습 (Self-Supervised Learning): 정답이 없는 수백만 장의 야외 식물 사진으로 Pre-training을 진행하여 도메인 지식을 선행 학습.
