🧠 3-2. 아키텍처 최적화: 5가지 모델 비교 분석
자체 CNN → GAP 경량화 → 전이학습 모델(ResNet50, MobileNetV3, EfficientNet-B0) 비교
Step 1. 전통적 CNN의 한계 발견 (Flatten 사용)
Flatten 층으로 인해 파라미터가 약 2,610만 개로 폭발적 증가.
50층짜리 ResNet50(약 2,350만 개)보다 무거워 모바일/엣지 디바이스 배포에 부적합함을 발견.
Step 2. 자체 경량화 달성 (GAP 도입) → 파라미터 98% 감소
Flatten을 최신 논문 기법인 GAP(Global Average Pooling)로 직접 교체.
파라미터 2,610만 개 → 54만 개(0.5M)로 98% 대폭 감소. 극강의 초경량 자체 모델 구축 성공.
Step 3. 다양한 전이학습 모델 비교 실험
자체 경량화 연구를 바탕으로 ResNet50(Residual Connection), MobileNetV3(Depthwise Conv), EfficientNet-B0(Compound Scaling) 3가지 SOTA 모델을 도입하여 성능과 효율을 비교 분석.

무엇이 문제인가?

  • 실험실 데이터(PlantVillage): 단색 배경에 잎사귀 하나만 깨끗하게 촬영된 이미지
  • 야외 데이터(PlantDoc): 흙, 잡초, 가지, 그림자 등 복잡한 배경이 뒤섞인 실제 환경 이미지
  • 모델이 실험실의 단색 배경 패턴에 완전히 과적합되어, 야외 환경을 전혀 이해하지 못하는 전형적인 도메인 시프트(Domain Shift) 현상 발생

🚨 핵심 발견: 실험실에서 아무리 높은 정확도를 달성해도, 실제 현장 환경과의 괴리(Domain Gap)를 해결하지 못하면 실무 적용이 불가능하다는 사실을 확인했습니다.

🔍 7. 원인 분석: Grad-CAM 시각화
AI가 대체 어디를 보고 판단하는지, 모델의 시선을 직접 시각화

Grad-CAM이란?

Gradient-weighted Class Activation Mapping — 딥러닝 모델이 예측할 때 이미지의 어느 부위에 집중했는지를 히트맵으로 시각화하는 설명 가능한 AI(XAI) 기법입니다.

분석 결과

  • 발견: AI는 잎사귀의 병변을 전혀 보지 않고 있었습니다.
  • 오류 원인: 우측 사진처럼 뒷배경의 흙과 나뭇가지에 붉게 시선이 집중(활성화)되며 엉뚱한 식물로 오진했습니다.
  • 예시: 실제 정답은 사과 흑성병(Apple Scab)인데, 뒷배경을 보고 복숭아 세균성 구멍병으로 오답을 내렸습니다.
Grad-CAM 히트맵: 잎이 아닌 흙과 배경에 시선(붉은색)이 쏠림

🔬 근본 원인 확정: 모델이 잎의 병변이 아닌 흙이나 배경을 보고 판단하는 치명적 과적합이 발견되었습니다. 이 문제를 해결하기 위해 3가지 가설을 세우고 검증 실험에 돌입했습니다.

🧪 8-1. 가설 1: 물리적 전처리 도입 실패
배경이 문제라면, AI(rembg)로 배경을 물리적으로 제거하면 해결될까?

실험 설계

  • 방법: rembg(AI 배경 제거 모델)를 사용하여 야외 데이터 전체(학습+테스트)의 배경을 제거(누끼 따기)
  • 기대: 배경 노이즈가 사라지면 모델이 잎사귀에만 집중할 것
결과 정확도
47.81%

실패 원인 분석

  • 배경을 강제로 날리면서 정답을 유추할 '환경적 문맥(Context)'까지 파괴됨
  • 질병으로 썩어 들어간 잎 끝부분(병변)까지 배경으로 인식되어 함께 훼손됨
V3 학습 곡선: 47%에서 수렴

❌ 결론: 물리적 전처리(배경 제거)는 환경 문맥을 상실시키고 병변을 훼손하여, 오히려 모델의 판단 근거를 약화시킵니다.

🧪 8-2. 가설 2: 혼합 도메인 학습 실패
기존 실험실 데이터와 야외 데이터를 섞어서 학습하면 해결될까?

실험 설계

  • 방법: 실험실 데이터 5,000장 + 야외 데이터 2,500장을 혼합하여 파인튜닝
  • 기대: 과거의 깨끗한 데이터를 섞으면 질병 패턴 망각을 방지할 수 있을 것
결과 정확도
47.01%

실패 원인 분석

  • 모델이 가혹한 야외 환경에 뼈를 깎으며 적응해야 할 시점에, 쉬운 데이터가 섞여 들어오면서 모델의 시선이 분산됨
  • 야외 환경 적응(Adaptation)에 대한 학습 집중도가 희석되어 안일해짐
V4 혼합 학습 곡선: 47%에서 한계

❌ 결론: 쉬운 데이터를 혼합하면 야외 환경 적응에 대한 모델의 시선이 분산되어, 도메인 적응에 오히려 역효과를 초래합니다.

🧪 8-3. 가설 3: 본질적 도메인 적응 성공
원본 야외 데이터만 투입 + 가혹한 증강 + 최신 수학적 최적화로 전이학습

🚀 10. 향후 연구 방향
도메인 적응 성능(80~90% 이상)을 목표로 한 고도화 전략
🤖
Vision Transformer (ViT) 도입
국소적 픽셀에 집착하는 CNN의 한계를 넘어, 글로벌 문맥에 Self-Attention하여 배경 노이즈를 무시하고 잎사귀 병변에만 집중하도록 유도
⚔️
적대적 도메인 적응 (DANN)
모델 내부에 기울기 반전 계층(Gradient Reversal Layer)을 추가하여, 실험실과 야외의 도메인 차이를 모델이 구별하지 못하도록 강제
📦
객체 탐지(YOLO) 파이프라인 결합
무식한 배경 제거 대신, 전체 사진에서 '식물의 잎사귀' 영역만 Bounding Box로 정확히 크롭한 뒤 분류를 수행하여 노이즈 차단
🧬
